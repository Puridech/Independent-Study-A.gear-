# n8n AI Automation Workflows

โปรเจกต์นี้เป็นการพัฒนา AI Automation Workflow ด้วย n8n ประกอบด้วย 2 workflows หลักที่ทำงานได้จริง ได้แก่ **แชทบอท AI ทั่วไปบน Telegram** และ **ระบบถาม-ตอบจากเอกสาร (Document Q&A Bot)**

---

## 1. Workflow: AI Chatbot
**ไฟล์:** `Telegram Bot Ai Genmini.json`

แชทบอท AI บน Telegram ที่ทำหน้าที่ตอบคำถามทั่วไป สามารถพูดคุยและตอบโต้ได้อย่างเป็นธรรมชาติ

### ส่วนประกอบหลัก (Nodes):
*(เรียงลำดับการทำงาน)*
1. **Telegram Trigger:** ทำหน้าที่ดักจับและรับข้อความเริ่มต้นที่ผู้ใช้พิมพ์ส่งเข้ามา
2. **Set (Edit Fields):** คัดกรองและดึงเฉพาะข้อมูลที่จำเป็น (เช่น `ChatID` และเนื้อหาข้อความ) เพื่อส่งต่อให้ระบบทำงาน
3. **AI Agent & Google Gemini Chat Model:** ทำหน้าที่ประมวลผลข้อความและคิดคำตอบ โดยใช้โมเดลของ Google Gemini
4. **Simple Memory:** ระบบความจำระยะสั้น ช่วยให้บอทจดจำบริบทและประวัติการสนทนาก่อนหน้าได้ (ตั้งค่าไว้ที่ 5-10 ข้อความ)
5. **Telegram (Send Message):** ส่งข้อความผลลัพธ์จาก AI กลับไปหาผู้ใช้งานทางแชทบน Telegram

## 2. Workflow: Document Q&A Bot
**ไฟล์:** `Document Q&A Bot.json`

ระบบแชทบอทถาม-ตอบจากเอกสาร (พัฒนาต่อยอดจาก Workflow 1) โดยผู้ใช้สามารถอัปโหลดไฟล์ แล้วให้ AI วิเคราะห์และตอบคำถามจากเนื้อหาในเอกสารนั้นๆ (ระบบ RAG)

### ส่วนประกอบหลัก (Nodes):

#### ส่วนที่ 1: การจัดการเอกสาร (File Upload Flow)
1. **On form submission (Form Trigger):** หน้าเว็บฟอร์ม (Webhook) สำหรับรับไฟล์เอกสารที่อัปโหลด
2. **HTTP Requests (Create Store / Upload / Import):** ชุดคำสั่ง API ที่ทำหน้าที่สร้างคลังเก็บข้อมูล (File Search Store), อัปโหลดไฟล์ขึ้น Google Gemini, และดึงไฟล์เข้าสู่คลังเอกสารเพื่อเตรียมให้ AI เข้าถึง

#### ส่วนที่ 2: ระบบถาม-ตอบ (Q&A Chat Flow)
1. **Telegram Trigger:** รับข้อความคำถามจากผู้ใช้งาน
2. **AI Agent (RAG Agent):** ตัวประมวลผลที่ถูกตั้งคำสั่ง (System Prompt) ให้ทำหน้าที่ค้นหาข้อมูลจากเอกสาร และต้องระบุแหล่งที่มา/เลขหน้าทุกครั้งที่ตอบ
3. **DataFile (HTTP Request Tool):** เครื่องมือพิเศษที่เชื่อมต่อกับโมเดล `gemini-2.5-flash` เพื่อทำการค้นหาข้อมูล (Search) เจาะจงเฉพาะในคลังเอกสารที่อัปโหลดไว้
4. **Simple Memory & Gemini Chat Model:** ทำหน้าที่ประมวลผลภาษาและจดจำบริบทการสนทนา
5. **Telegram (Send Message):** สรุปข้อมูลที่ได้จากเอกสาร และส่งเป็นข้อความตอบกลับหาผู้ใช้

## 3. Workflow: Document Q&A Bot and General AI
**ไฟล์:** `Document Q&A Bot and General AI.json`

Workflow นี้นำการทำงานของ Workflow 1 และ 2 มารวมกัน เพื่อให้บอทมีความฉลาดและจัดการคำถามได้ยืดหยุ่นมากขึ้น

### จุดที่เพิ่มเข้ามา:
* **Node If (Router):** ใช้เป็นตัวแยกแยะเส้นทางการทำงาน
  * หากข้อความที่พิมพ์มามีคีย์เวิร์ดเช่น *"เอกสาร, ไฟล์, สรุป, PDF"* ระบบจะส่งให้ **AI Agent (Document)** ทำงาน (รูปแบบเดียวกับ Q&A Bot ดั้งเดิม)
  * หากเป็นคำถามพูดคุยทั่วไป ระบบจะส่งให้ **AI Agent (General)** เป็นคนตอบแทน เพื่อความรวดเร็วและเป็นธรรมชาติ
